1. Class ModelStore - хранение структуры 3D редактора
   - связь реализация: переопределение метода notifyChange интерфейса (комплементация) iModelChanger
   - связь ассоциация: использование в качестве поля в классе ModelStore поля iModelChangedObserver в виде списка и
     в конструктор не добавляется, так как мощность от интерфейса iModelChangedObserver 0...n.
   - связь  комплементация: создание конструктора с созданием объектов (PoligonalModel, Camera, Flash,Scene )
   - в конструктор также передается в качестве аргумента Texture, необходимая для создания объекта PoligonalModel
   - метод получения сцены по номеру запрашиваемой сцены  public Scene getScene(int sceneNumber)
2. Class Flash - свойства источника света для 3D редактора
    - отношения идут от класса к другим классам, поэтому в нем указываются только поля и методы
    - переменные типа Point3D и Angle3D являются бинарными переменными, то есть имеют одно значение
3. Class Camera - расположение камеры для 3D редактора
    - поток данных идет от класса, поэтому в нем указываются только поля и методы
    - переменные типа Point3D и Angle3D являются бинарными переменными, то есть имеют одно значение
4. Class PoligonalModel - полигональное моделирование обекта, модель полигона
    - указываются бинарные поля Poligon poligons и Texture textures
    - создание конструктора для приема данных, так как поток данных идет от Texture и Poligon 
    - поле poligons создается непосредственно в теле конструктора (отношение композиция) и 
     создается только вместе с классом  PoligonalModel и при его уничтожении, этот объект тоже
     уничтожится.
    - поле textures получается из переданного аргумента конструктора (отношение агрегация) и передается из внешнего
      источника как ссылка 
5. Class Texture - текстура (сетка) для создания  PoligonalModel
6. Class Poligon - полигон  для создания  PoligonalModel
   - поле Point3D points указываем в виде списка, так как есть мощность 1..*  (означает, что должна быть хотя бы одна точка)
   - создается конструктор для добавления в список поля Point3D points значения точки
7. Class Scene - набор необходимых элементов для создания модели в 3D редакторе
   - поля указываем в виде списков, так как есть мощность 1..* и 0..*
   - создание конструктора для приема данных, так как направление данных идет к классу Scena
   - в конструкторе прописываем поля через аргументы констрктора, так как отношение агрегация, поле Flash можно не указываем, 
     так как мощность 0..* (Flash может отсутствоать)
   - создаем методы c использованием дженериков
8. Interface iModelChanger - вносит изменение в модель пользуясь хранилищем ModelStore
   - отношение - реализация - пишется сигнатура метода NotifyChang
   - в классе ModelStore уже переопределяется метод интерфейса
9. Interface iModelChangedObserver - отслеживание за изменениями  пользуясь хранилищем ModelStore
    - отношение - ассоциация - пишется сигнатура метода - ApplyUpdateMode()
    - в классе ModelStore метод ApplyUpdateMode() не реализован. Включается класс iModelChangedObserver в класс ModelStore
